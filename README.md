# سامانه طرح کنترل محصول و ثبت نتایج بازرسی

اپلیکیشن وب چندکاربره برای تعریف طرح‌های کنترل کیفیت (Control Plan) و ثبت داده‌های بازرسی حین تولید، با اعتبارسنجی خودکار پاس/فیل بر اساس تلرانس تعریف‌شده.

## معماری

- **بک‌اند:** Node.js + Express
- **دیتابیس:** SQLite — با استفاده از ماژول داخلی `node:sqlite` (نیازی به نصب یا کامپایل هیچ بسته جانبی برای دیتابیس نیست)
- **احراز هویت:** JWT، با سه نقش: `admin` (مدیر سیستم)، `qc_manager` (سرپرست کیفیت)، `inspector` (بازرس)
- **فرانت‌اند:** یک صفحه HTML/JS ساده که مستقیماً توسط همین سرور Express سرو می‌شود — نیازی به سرور جداگانه یا build نیست

داده‌ها در یک فایل واحد (`server/data/qc.db`) ذخیره می‌شوند. برای ۵ تا ۲۰ کاربر همزمان این معماری کاملاً کافی و پایدار است.

## پیش‌نیاز سرور

- **Node.js نسخه ۲۲.۵ یا بالاتر** (به دلیل استفاده از `node:sqlite`). نسخه را با دستور زیر بررسی کنید:
  ```
  node -v
  ```
  اگر نسخه سرور شما قدیمی‌تر است، یا Node را به‌روزرسانی کنید یا این موضوع را اطلاع دهید تا نسخه‌ای با `better-sqlite3` (سازگار با نسخه‌های قدیمی‌تر Node، اما نیازمند ابزار کامپایل native) آماده شود.

## روش نصب اول: با Docker (پیشنهادی)

نیاز به نصب Node روی سرور نیست؛ فقط Docker و Docker Compose کافی است.

```bash
# ۱- وارد پوشه پروژه شوید
cd qc-app

# ۲- فایل env بسازید و JWT_SECRET را تغییر دهید
cp .env.example .env
nano .env

# ۳- بیلد و اجرا
docker compose up -d --build

# ۴- ساخت اولین کاربر مدیر (فقط بار اول)
docker compose exec qc-app node server/seed-admin.js -- --username=admin --password="یک-رمز-قوی" --name="نام مدیر"
```

بعد از این مراحل، سامانه روی آدرس `http://IP-سرور:4000` در دسترس است.

داده‌ها در یک Docker volume به نام `qc-data` ذخیره می‌شوند و با ری‌استارت یا آپدیت کانتینر از بین نمی‌روند.

### به‌روزرسانی بعدی
```bash
docker compose down
docker compose up -d --build
```

## روش نصب دوم: بدون Docker (مستقیم روی سرور)

```bash
# ۱- وارد پوشه سرور شوید
cd qc-app/server

# ۲- نصب وابستگی‌ها (بدون نیاز به کامپایلر — سریع و بدون مشکل)
npm install

# ۳- فایل env بسازید
cp .env.example .env
nano .env
# مقدار JWT_SECRET را به یک رشته تصادفی طولانی تغییر دهید

# ۴- ساخت اولین کاربر مدیر
npm run seed-admin -- --username=admin --password="یک-رمز-قوی" --name="نام مدیر"

# ۵- اجرای سرور
npm start
```

سرور به‌صورت پیش‌فرض روی پورت ۴۰۰۰ بالا می‌آید (قابل تغییر در `.env`).

### اجرای دائمی (پیشنهاد برای production)
برای اینکه سرور بعد از قطع ترمینال یا ری‌استارت سرور هم روشن بماند، از `pm2` یا یک سرویس `systemd` استفاده کنید:

```bash
npm install -g pm2
pm2 start server.js --name qc-app
pm2 save
pm2 startup
```

## مدیریت کاربران

- اولین کاربر (مدیر سیستم) با اسکریپت `seed-admin.js` ساخته می‌شود.
- کاربران بعدی از داخل خود اپلیکیشن، در تب **«مدیریت کاربران»** (فقط برای نقش مدیر سیستم قابل مشاهده است) اضافه می‌شوند.
- سه نقش:
  - **مدیر سیستم (admin):** دسترسی کامل، مدیریت کاربران
  - **سرپرست کیفیت (qc_manager):** تعریف/ویرایش/حذف طرح‌های کنترل + ثبت داده
  - **بازرس (inspector):** فقط ثبت داده بازرسی و مشاهده گزارش‌ها

## پشتیبان‌گیری (Backup)

کل داده‌های سامانه در یک فایل است:
```
server/data/qc.db
```
(در حالت Docker، این فایل داخل volume به نام `qc-data` قرار دارد.)

برای پشتیبان‌گیری، کافی است این فایل (همراه با فایل‌های کمکی `qc.db-wal` و `qc.db-shm` در صورت وجود) را به‌صورت دوره‌ای کپی کنید. پیشنهاد می‌شود یک job روزانه (cron) برای این کار تنظیم شود.

## نکات امنیتی برای تیم IT

- مقدار `JWT_SECRET` را حتماً به یک رشته تصادفی و طولانی تغییر دهید و آن را در اختیار کسی قرار ندهید.
- اگر سامانه از خارج شبکه داخلی هم در دسترس باشد، حتماً پشت HTTPS (مثلاً با Nginx reverse proxy) قرار بگیرد.
- رمزهای عبور کاربران با bcrypt هش می‌شوند و به‌صورت متن ساده در دیتابیس ذخیره نمی‌شوند.
